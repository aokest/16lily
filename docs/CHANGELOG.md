# 变更日志 (Changelog)

> **项目**: 智能商机跟进及业绩统计系统
> **维护者**: Trae AI Assistant

## v1.2.1 (2026-01-08)

### 🐞 修复与稳定性 (Fixes & Stability)
- **项目详情页白屏修复**:
    - 修复了 `MainLayout.vue` 中 `@/api` 路径别名在 Docker 环境下解析失败导致的关键资源加载错误。
    - 优化了 `ProjectBoard.vue` 的错误处理机制，增加全局错误提示 (ElMessage)，避免静默失败导致的白屏。
    - 修复了软删除项目导致的前端 404 路由问题。
- **AI 润色功能修复**:
    - 修复了 AI 配置未激活导致的服务静默失败。
    - 解决了“润色后内容无变化”的问题，通过强制重写指令优化了 Prompt。

### ✨ 功能增强 (Enhancements)
- **分场景 AI 润色**:
    - 新增基于用户角色 (Role-Based) 的提示词策略。
    - **研发 (RND)**: 侧重技术术语、进度量化与风险揭示。
    - **销售 (SALES)**: 侧重商机阶段、回款预测与客户触达。
    - **管理 (MANAGEMENT)**: 侧重资源协调与里程碑风险。

## v1.2.0 (2026-01-08)

### 🚀 核心功能迭代
- **项目管理**:
    - 修复了 `ProjectViewSet` 导入漏洞导致的后端 500 错误。
    - 增强了项目软删除逻辑与回收站机制。
- **通知系统**:
    - 集成 `clean_notifications` 命令，解决脏数据问题。
- **AI 配置管理**:
    - 修复了配置隔离性问题，确保多用户环境下的配置安全。

## v1.1.0 (2026-01-06)

### ✨ 核心功能
- **职级体系重构**: 拆分“高级副总裁/助理总裁”为两个独立职级，适配实际组织架构。
- **岗位管理增强**: 修复新增岗位按钮失效问题，优化初始化默认岗位逻辑。
- **权限与授权**: 修复助理同权设置时的 405 错误，优化 AI 助手代理授权流程。

### 🚀 运维与部署
- **一键部署 2.0**: 重构 `deploy.sh`，支持全量清理云端环境、自动初始化数据库、自动创建超级管理员。
- **数据同步**: 建立核心架构种子数据同步机制，确保本地与云端组织架构一致。
- **配置优化**: 针对 2C/2G 云服务器优化 Docker 内存分配与数据库连接池。

### 🐞 修复
- 修复业绩指标拆分时的唯一性约束报错。
- 修复退出登录后部分 Session 残留问题。
- 修复云端环境菜单显示不全的问题。

## v0.7.0-beta (2025-12-10)

### ✅ 审批与活动中心
- 新增 `ApprovalRequest` 统一审批模型，自动路由到部门经理；支持管理员越权。
- 新增 `ActivityLog` 统一记录事件；审批通过/驳回写入活动流并在大屏显示。

### 📣 社媒粉丝审批
- `SocialMediaStats` 增加状态与维护人，创建即生成审批请求。
- 开放 API：`/api/social-stats/`。

### 🧭 导航与报表
- 新增 `CRM 首页` 与双向入口；大屏右上角加入“CRM首页/业绩报表”。
- 新增业绩报表页面 `/reports/performance`。

### 🧪 测试体系
- 建立 `tests/` 目录与模板；新增 `verify_approvals` 与 `verify_socialmedia` 管理命令。
- 产出审批测试报告 `tests/reports/2025-12-10-approvals.md`。

### 🚀 架构升级 (Architecture)
- **环境恢复**: 成功修复 Docker 守护进程连接问题，将数据库配置从 SQLite 临时模式切回 Docker/PostgreSQL。
- **信号重构 (Refactor)**: 将 "商机移交" 的业务逻辑从 `admin.py` 迁移至 `core/signals.py`。
    - 优势：解耦了 Admin 界面与业务逻辑，确保未来通过 API (Vue前端) 也可以触发同样的移交审批流程。
- **前端兼容性**: 修复 Vite/Tailwind v4 在 Docker 环境下的网络连接与构建配置问题。

### 📋 规划更新 (Roadmap)
- **Phase 3启动**: 正式确立 "去 Jazzmin 化" 战略。
    - 目标：逐步将核心 CRM 功能（商机管理、跟进记录）迁移至 Vue 3 Dashboard。
    - 现状：后端已就绪 (DRF + Signals)，前端已就绪 (Vue + Element Plus)。
