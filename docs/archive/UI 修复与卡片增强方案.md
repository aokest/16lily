# UI 修复与卡片增强方案

针对您的 4 点反馈，我制定了以下修复与增强计划：

## 1. 个人中心待办列表修复 (PersonalCenter.vue)
**问题**: 待办事项列表项紧贴左右边框。
**修复方案**:
- 给 `<ul>` 容器增加 `px-6` (24px) 内边距。
- 调整 `<li>` 的 `-mx-6` 和 `px-6` 逻辑，确保 Hover 效果铺满整行但内容保持缩进。

## 2. 业绩报表修复 (Performance.vue)
**问题**: 右侧按钮不可见，个人目标卡片消失。
**修复方案**:
- **修复按钮**: 将所有 `bg-pomegranate-500` 类替换为 `style="background-color: #D64045;"`，确保颜色正确渲染。
- **恢复个人目标**: 在 Tab 1 (报表概览) 中，重新添加“个人目标及完成情况”卡片（目前代码中确实缺失了这一块，我将从 PersonalCenter 中复用逻辑或重新实现）。
- **Mock 数据**: 在 `fetchData` 中添加逻辑，如果 API 返回空数据（全0），则填充演示用的 Mock 数据，确保演示效果。

## 3. 卡片编辑器布局优化 (CardEditor.vue)
**问题**: 卡片长宽比变形，无法自由调整左右比例。
**修复方案**:
- **锁定 16:9**: 给卡片预览容器添加 `aspect-ratio: 16/9` 样式。
- **缩放适配 (Transform Scale)**: 实现一个 `AutoFit` 容器，计算当前可用宽度与目标宽度（如 1280px）的比例，使用 `transform: scale()` 进行等比缩放，确保卡片内容（字体、间距）在任何屏幕下都保持原样，只是整体缩小/放大。
- **可拖拽分割线**: 实现一个 `Resizer` 组件，允许用户拖动调整左右分栏的宽度比例。

## 4. 卡片配色自定义 (CardEditor.vue)
**问题**: 样式太普通，不支持换色。
**修复方案**:
- **主题切换器**: 在编辑器顶部工具栏增加“主题设置”按钮。
- **预设主题**: 内置 5 套高级配色方案（如：深邃黑金、商务蓝灰、活力橙白等）。
- **实时预览**: 绑定样式变量到 CSS Variable，实现点击即换色。
