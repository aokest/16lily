# 🚀 后续系统研发与迭代计划

鉴于基础架构（个人中心、组织架构、数据模型）已逐步稳定，接下来的开发将重点转向**业务流程自动化**、**数据智能分析**以及**权限精细化控制**。

## Phase 1: 核心业务流程完善 (Current Focus)
### 1. 审批工作流引擎 (Workflow Engine)
- **目标**：实现商机、赛事、市场活动的标准化审批流程。
- **内容**：
    - [ ] **商机立项审批**：销售提交 -> 部门经理审批 -> 财务确认。
    - [ ] **赛事/活动预算审批**：自动根据规模触发不同层级的审批链。
    - [ ] **审批中心**：在“个人中心”增加“我的审批”卡片，集中处理待办事项。
    - [ ] **状态自动流转**：审批通过后，系统自动更新项目状态并发送通知。

### 2. 权限与数据隔离 (RBAC Enhancement)
- **目标**：基于新的 `DepartmentModel` 实现更细粒度的数据权限。
- **内容**：
    - [ ] **部门隔离**：销售只能看到本部门或下属的商机；部门经理可看全部门。
    - [ ] **跨部门协作**：允许手动添加“协作成员”，授予特定项目的读写权限。
    - [ ] **字段级权限**：例如，普通销售不可见“毛利”字段，仅财务和总监可见。

## Phase 2: AI 智能化集成 (Smart Features)
### 3. AI 助手 "粒粒" (LiLi AI)
- **目标**：利用大模型减少手工录入成本，提供决策辅助。
- **内容**：
    - [ ] **商机智能解析**：粘贴微信聊天记录或邮件内容，AI 自动提取客户名、需求、预算等信息填入表单。
    - [ ] **周报自动生成**：一键汇总本周的跟进记录、新增商机，生成格式化的周报草稿。
    - [ ] **赛事Excel智能清洗**：增强当前的导入功能，利用 AI 自动修正 Excel 中的非标准数据（如模糊的日期格式、错别字）。

## Phase 3: 数据可视化与大屏 (Analytics & BI)
### 4. 销售漏斗与报表
- **目标**：提供管理层决策支持数据。
- **内容**：
    - [ ] **销售漏斗图**：可视化展示从“线索”到“赢单”的转化率。
    - [ ] **业绩预测 (Forecast)**：基于当前商机阶段和历史转化率，预测本季度营收。
    - [ ] **大屏战报优化**：美化现有大屏，增加实时滚动的签约喜报和排行榜。

## Phase 4: 移动端适配与通知 (Mobile & Notification)
### 5. 企业微信/钉钉集成
- **目标**：实现移动办公。
- **内容**：
    - [ ] **消息推送**：审批提醒、商机变更提醒推送到企微/钉钉。
    - [ ] **免登跳转**：从消息卡片直接点击进入系统详情页。
    - [ ] **移动端H5适配**：优化 Jazzmin 在手机浏览器上的显示体验。
