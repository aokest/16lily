# UI/UX 修复与优化方案

根据您的反馈，我将对以下三个核心问题进行针对性修复。

## 1. 侧边栏清理 (MainLayout.vue)
**问题**: 侧边栏的“小字”（分组标题）和“系统设置”标题无法点击且造成困惑。
**修复方案**:
- **移除分组标题渲染**: 删除侧边栏中渲染 `group.title` 的代码块（即“个人中心”、“业务管理”等小字）。
- **保留核心功能**: 仅移除视觉上的标题，保留“用户管理”、“AI配置”等子菜单项（除非您确认要彻底删除这些功能，目前按仅优化视觉处理）。
- **效果**: 侧边栏将变为纯图标+文字的列表，更加简洁。

## 2. 项目列表卡片布局修复 (ProjectList.vue)
**问题**: 项目卡片中的标签（状态/名称）导致布局变形。
**修复方案**:
- **增强弹性布局**: 在卡片标题容器添加 `min-w-0`，允许标题在空间不足时自动截断，防止撑开容器。
- **固定标签宽度**: 对状态标签（Status Tag）添加 `flex-shrink-0` 和 `whitespace-nowrap`，确保它不会被压缩或换行，同时不会挤压标题。
- **截断优化**: 确保长标题和长编号都能正确触发 `line-clamp` 或 `truncate`。

## 3. 卡片编辑器笔记本适配 (CardEditor.vue)
**问题**: 在笔记本屏幕上，弹窗显示不全（被截断）。
**修复方案**:
- **调整高度策略**: 将弹窗的最大高度从 `max-h-[90vh]` 调整为 `h-[85vh]`（固定高度），预留更多上下边距以适应任务栏。
- **优化内部滚动**: 强制左右分栏（预览区和编辑区）的高度为 `h-full`，并确保 `overflow-y-auto` 正确生效，使得内容在弹窗内部滚动，而不是撑大弹窗。
- **响应式调整**: 检查 `lg:` 断点，确保在较矮的屏幕上也能正确布局。
