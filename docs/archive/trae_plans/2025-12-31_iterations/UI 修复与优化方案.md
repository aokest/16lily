# UI 修复与优化方案 (修订版)

根据您的最新确认，调整后的方案如下：

## 1. 修复卡片编辑页“上一页/下一页”失效 (StandaloneCardEditor.vue)
**问题**: 独立窗口模式下丢失了卡片列表上下文，无法切换。
**修复方案**:
- **获取上下文**: 在加载卡片时，同时请求该项目下所有卡片的 ID 列表（轻量级请求）。
- **计算逻辑**: 在前端计算当前卡片的 index，从而判断 `hasPrev` / `hasNext`。
- **导航实现**: 点击上一页/下一页时，直接修改路由参数 ID，触发组件重新加载数据。

## 2. 统一 UI 风格 & 修复个人中心 (PersonalCenter.vue)
**问题**: 待办事项文字贴边难看，整体风格不统一。
**修复方案**:
- **重构待办列表**: 将 HTML 硬编码列表改为 `v-for` 渲染，数据源放在 `<script>` 中。
- **修复间距**: 给列表容器添加 `px-6` (24px) 内边距，增加 `py-4` 行间距，解决贴边问题。
- **文档记录**: 创建 `DEV_NOTES.md`，记录此处为 Mock 数据，方便后端接入。

## 3. 业绩报表视觉升级 (Performance.vue)
**问题**: 视觉效果 Low，缺乏高级感。
**修复方案**:
- **移除 Element Plus 容器**: 废弃 `el-card`，改用自定义的白色圆角容器 (`bg-white rounded-2xl shadow-sm`)。
- **引入品牌色**: 关键 KPI 数字使用 Pomegranate Red (#D64045) 和 Gold (#D4AF37) 进行强调。
- **现代化排版**: 增加留白，使用更细的分割线，优化表格头部样式。

## 4. 战报大屏导航确认
**确认项**: 战报大屏是一个独立页面 (`/`)。
**动作**: 确保 CRM 首页顶部导航栏的“战报大屏”按钮，点击后通过 `window.open('/', '_blank')` 打开新标签页，且该页面有明确的“返回 CRM”按钮（如果需要）。
