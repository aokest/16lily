# AI 使用说明与提示词

## 1. AI 配置指南
本系统支持兼容 OpenAI 接口格式的所有大模型服务商。

### 推荐配置
- **OpenAI**:
  - Base URL: `https://api.openai.com/v1`
  - Model: `gpt-4o-mini` (性价比高) 或 `gpt-4o`
- **DeepSeek (深度求索)**:
  - Base URL: `https://api.deepseek.com`
  - Model: `deepseek-chat`
- **Moonshot (Kimi)**:
  - Base URL: `https://api.moonshot.cn/v1`
  - Model: `moonshot-v1-8k`
- **阿里通义千问**:
  - Base URL: `https://dashscope.aliyuncs.com/compatible-mode/v1`
  - Model: `qwen-plus`

### 常见错误排查
- **400 Model Not Exist**: 
  - 检查 Base URL 是否与服务商对应。
  - 检查 Model 名称是否拼写正确（如 `gpt-4` vs `gpt-4-turbo`，`deepseek-chat` vs `deepseek-v3`）。
- **401 Unauthorized**: API Key 错误或过期。

## 2. 系统提示词 (System Prompt)
这是写入代码中的默认提示词，用于指导 AI 进行卡片内容润色。

> 你是一位国内顶尖的网络安全咨询顾问（Security Consultant），服务于政府、金融、能源等关键基础设施客户。
> 请对以下项目售前卡片的内容进行深度润色和优化。
> 
> 核心目标：
> 提升内容的专业性、商业价值和说服力，使其听起来像一份高价值的安全服务方案，而不仅仅是简单的任务清单。
> 
> 具体要求：
> 1. **角色带入**：使用网络安全行业的专业术语（如：风险闭环、态势感知、攻击面管理、主动防御、合规性、APT攻击链等）。
> 2. **价值导向**：在“项目目标”和“现状分析”中，不仅描述问题，更要强调风险的严重性（如数据泄露、监管合规、业务连续性）和解决后的安全收益。
> 3. **内容升级**：
>    - “项目内容”应体现方法论（如 PDCA、PDR 模型）或标准化服务流程。
>    - “现状分析”要切中痛点，体现对客户安全态势的深刻理解。
> 4. **格式约束**：
>    - 必须严格保持原有的“键：值”格式（如“项目名称：...”，“项目目标：...”等），确保系统能解析。
>    - 保持段落分明，分点描述时使用标准序号或 bullet points。
> 5. **长度控制**：
>    - 项目名称：<20字，精炼且高大上。
>    - 项目目标/现状分析：<80字，字字珠玑。
>    - 项目内容：分点清晰，逻辑严密。

## 3. 下一个 AI 交接提示词 (Handover Prompt)
**任务：接手“项目卡片及推进表生成器”项目开发**

**当前项目状态 (2025-12-25)：**
- **核心功能**：单页应用 (SPA)，基于 React (CDN) + TailwindCSS。
- **关键文件**：
  - `cardv8.html`: 核心编辑器，v9.2 版本（高度稳定）。使用了统一组件 `<CardSection>` 和手动防御性 `Icon` 渲染。**严禁随意改动 DOM 结构**，否则会导致样式塌陷或点击失效。
  - `project_timeline.html`: 时间表页面，支持自动从 API 拉取数据，且支持**手动编辑**时间。
  - `server.py`: Python Flask 后端，提供 `/api/cards` 接口，负责数据读写。
- **已知痛点与防御机制**：
  - **白屏陷阱**：Lucide 图标库与 React 渲染冲突。**切勿使用 `lucide.createIcons()`**，必须使用当前的 `toPascalCase` + `svg` 字符串注入方案。
  - **点击失效陷阱**：卡片模块必须使用 `e.stopPropagation()` 和 `e.preventDefault()` 严防事件冒泡和默认跳转。
  - **样式陷阱**：卡片布局采用 CSS Grid 12 列系统，修改布局时需极其小心 Flexbox 和 Grid 的嵌套关系。

**你的任务准则**：
1.  **先读文档**：开始工作前，务必阅读 `docs/04-开发进度.md` 中的“2025-12-25 深度复盘”，了解之前的“血泪史”。
2.  **组件化思维**：不要写 CSS 补丁。如果发现样式问题，去调整 `<CardSection>` 组件，确保 9 个模块一致性。
3.  **验证第一**：任何交互修改（特别是点击、选中），必须在修改后立即验证“是否会导致页面跳转”或“无法选中”。

**启动命令**：
1.  确保 Docker 容器运行中：`docker ps`
2.  访问：`http://localhost:18889/index.html`
