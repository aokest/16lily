## 模块一：样式预设保存/加载
- 目标：让用户将当前样式参数（fontSize、lineHeight、titleSize、borderWidth、主题色）保存为命名预设，并可一键切换/删除。
- 实施步骤：
  1. 在 CardEditor 工具栏“样式”菜单底部新增“保存为预设”输入框与保存按钮；右侧新增“样式预设”下拉列表。
  2. 预设数据结构：`{ id, name, fontSize, lineHeight, titleSize, borderWidth, theme }`；先存 localStorage（key: `crm_card_style_presets`），后续拓展为后端接口。
  3. 应用预设：选择后将参数写入 `cardConfig` 与 `currentTheme`，预览即时更新。
  4. 删除预设：在列表项右侧提供删除图标，操作后更新存储。
- 交付与验证：
  - 手工：保存/切换/删除预设全流程；刷新后预设仍在。
  - 自动化：用 webapp-testing 验证菜单交互与参数应用。

## 模块二：个人中心真实数据对接
- 目标：替换现有 Mock，接入后端接口（待办、项目、消息）。
- 实施步骤：
  1. Todo：GET `/todotasks/`（分页）；映射到 `approvalList` 与“待推进”；增加加载态与空态。
  2. Projects：GET `/projects/`；映射到 `projectList`；点击进入详情。
  3. Messages：GET `/messages/`（如无统一接口，沿用 localStorage + 后端发布入口）；滚动加载与“全部已读”。
- 交付与验证：
  - 手工：接口连通，数据渲染正确；无权限或空态可见。
  - 自动化：基本列表加载与详情跳转。

## 模块三：业绩报表五项指标聚合
- 目标：展示部门/个人两套指标（商机池、新签合同、回款、毛利、确认收入）。
- 实施步骤：
  1. 后端：新增聚合 API `/reports/metrics?scope=dept|user&date_range=...`，返回五项指标的数量与金额。
  2. 前端：在 `Performance.vue` 增加标签切换（部门/个人），绑定 API 数据；统一卡片视觉与单位格式（¥）。
- 交付与验证：
  - 手工：切换标签数据同步；日期范围选择有效。
  - 自动化：调用两个 scope 的断言与展示。

## 模块四：自动化测试落地（webapp-testing 技能）
- 目标：为关键交互建立 Playwright 脚本，形成可回归的最小集。
- 实施步骤：
  1. 样式菜单：点击出现、滑块联动截图。
  2. 输出物布局：定位右列第三行高度与左列底线对齐。
  3. 通知铃铛：默认数据注入、倒序展示最近 10 条、红点联动。
- 交付与验证：
  - 自动化：脚本执行通过；生成截图归档到 `docs/test_artifacts/`。

## 模块五：导出与主题扩展
- 目标：支持卡片导出 PNG/PPT，并扩展主题库的预设。
- 实施步骤：
  1. PNG：在 CardEditor 添加“导出为图片”，使用 `html2canvas`（若项目已有依赖，否则改用后端截图服务）。
  2. PPT：借助 `pptx` 技能生成基础模板（标题/正文/分隔线），并填充当前样式参数。
  3. 主题扩展：在 `theme-factory` 基础上整理 5 套常用主题（深色奢华、商务蓝灰、森林绿、海洋蓝、日落橙）。
- 交付与验证：
  - 手工：导出文件可用；主题切换与样式参数兼容。

## 预计风控
- 接口兼容性：后端未提供时先 localStorage 方案，接口就绪后无缝替换。
- 依赖管理：导出 PNG 不引入过重依赖，优先复用现有工具或后端服务。
- 测试维护：自动化脚本以“最小可回归集”为原则，避免过度耦合。
