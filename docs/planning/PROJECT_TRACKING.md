本文档旨在为所有参与者（包括人类开发者与AI助手）提供一个统一的视角，明确“我们为什么做这个”、“现在做到哪了”以及“接下来要去哪里”。

---

## 1. 项目初衷 (Vision)

本项目旨在为团队打造一个集**商机管理、业绩追踪、全员协作、激励展示**于一体的数字化平台。
我们希望打破部门壁垒，通过现代化、可视化的手段，让每一个角色的贡献都被看见，让团队目标清晰可见。这不是一个简单的Excel替代品，而是一个能够**提振士气**、**驱动增长**的作战指挥中心。

---

## 2. 核心目标 (Goals)

1.  **全员协同 (Collaboration)**: 覆盖销售、售前、POC、研发、实验室等多角色，明确各阶段职责，消除信息孤岛。
2.  **流程规范 (Standardization)**: 实现商机从接触到售后的全生命周期管理，确保数据准确、可追溯。
3.  **激励人心 (Motivation)**: 通过“大屏看板”和“音效提示”，实时展示胜利成果（如赢单战报），提振团队士气。
4.  **数据驱动 (Data-Driven)**: 提供多维度的筛选、统计和财务分析，辅助管理层进行精准决策。

## 4. 项目进展 (Progress Log)

详细日志请参考 [DEV_LOG.md](../DEV_LOG.md)。

### 🟢 已完成 (Done)
- **基础架构**: Django + PostgreSQL + Docker 环境搭建完毕。
- **后台管理**: 基于 `Jazzmin` 的定制化后台，实现了“石榴红+金”主题，侧边栏分组优化。
- **AI 智能填单**: 
    - 实现了 `Chat-to-Form` 架构，支持 DeepSeek/Ollama。
    - 解决了小模型 (Qwen3:8b) JSON 解析不稳定的问题 (One-Shot Prompt + 鲁棒清洗逻辑)。
    - 增加了 `PromptTemplate` 动态管理。
- **文档体系**: 建立了完整的文档驱动开发 (DDD) 流程，包含 `AI_COLLABORATION_GUIDE.md`。

### 🟡 进行中 (In Progress)

- **审批框架**: `ApprovalRequest` 已上线，完善前端审批视图与流程。
- **报表深化**: `/reports/performance` 增加时间区间与维度切换。

## 7. 需求池 (Backlog)

- 工种序列/岗位/职级严格联动：
  - 选择 `M` 管理序列 → 仅显示管理职级；岗位名称与职级一致；不出现专员/技术岗位。
  - 选择 `S` 销售序列 → 仅显示销售职级；岗位名称与职级一致；不出现媒介/UI等技术岗位。
  - 选择 `P` 技术序列 → 仅显示技术职级；岗位从技术岗位库选择。
  - 目标：在新 Admin SPA 中通过前端动态过滤与后端校验双层实现（当前 Django Admin 已做后端校验，前端交互将在迁移后完成）。

- 导航重构（Admin SPA 首批）：
  - 一级分类：
    - 经营中心：业绩报表、KPI、趋势与漏斗、大屏战报
    - 销售管理：商机列表/看板、审批中心、活动/赛事
    - 客户关系：客户、联系人、分群/标签
    - 社媒管理：账号、粉丝、管理员、历史（整合为一个入口）
    - AI中枢：模型配置、提示词、对话窗
    - 系统配置：组织架构、权限与审计、令牌/集成
  - 目标：消除重复菜单（如社媒管理员/历史/账号分散），统一为模块首页 + 子页导航；前后端路由与文案一致。
