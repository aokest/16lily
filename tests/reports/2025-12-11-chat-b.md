# 测试报告：AI对话接口 Phase B（列表与更新）

## 概述
- 模块/主题：扩展 `POST /api/chat/` 支持商机/客户列表与客户更新
- 日期：2025-12-11
- 负责人：Trae AI

## 测试方法
- 账号：`manager_sales`（密码 `123456`）
- 获取token：`POST /api/api-token-auth/`

## 用例与POC
- 商机列表（状态筛选/限制返回条数）：
  - `POST /api/chat/` `{ "intent":"list", "entity":"opportunity", "filters": { "status":"ACTIVE", "limit": 3 } }`
- 客户列表（行业筛选/限制返回条数）：
  - `POST /api/chat/` `{ "intent":"list", "entity":"customer", "filters": { "industry":"教育", "limit": 5 } }`
- 更新客户：
  - `POST /api/chat/` `{ "intent":"update", "entity":"customer", "fields": { "id": 9, "industry":"教育科技", "address":"北京" } }`

## 实际结果
- 商机列表返回 200，按筛选与限制生效。
- 客户列表返回 200，按筛选与限制生效。
- 更新客户返回 200，并在活动流中记录“更新客户”。

## 回归
- `GET /api/dashboard/activities/` 出现“创建客户/更新客户/审批通过”记录，排序与展示正常。

## 结论
- 通过；将继续扩展 `update/list` 到活动/赛事以及更丰富的过滤字段，并集成前端对话窗。
